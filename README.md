# OlcRTC Manager Panel

Web control panel for [olcrtc](https://github.com/openlibrecommunity/olcrtc).

It runs one or more `olcrtc` server instances, serves per-client OlcBox links, and provides a small web UI for managing clients.

## Features

- Manage olcrtc clients from a web panel.
- Create clients with a dedicated room and encryption key.
- Edit client carrier, transport, DNS, and display name.
- Copy raw OlcBox URI, for example `olcrtc://wbstream?datachannel@room#key%client$name`.
- Serve plain text subscription at `/<client-id>`.
- Reload changed olcrtc instances without restarting unchanged ones.
- Embedded React/Tailwind UI in a single Go binary.
- Basic Auth for admin API/UI via environment variables.

## Status

Early MVP. Keep the panel behind HTTPS and authentication. Do not expose it directly without a reverse proxy.

## Build

Requirements:

- Go 1.24+
- Node.js 20+
- pnpm 9+

```sh
pnpm install
pnpm build
go test ./...
go build -o olcrtc-manager ./cmd/olcrtc-manager
```

## Run

```sh
export OLCRTC_PATH=/usr/local/bin/olcrtc
export OLCRTC_MANAGER_ADDR=127.0.0.1
export OLCRTC_MANAGER_USER=admin
export OLCRTC_MANAGER_PASS='change-me'

./olcrtc-manager -config ./examples/config.example.json
```

Open:

```text
http://127.0.0.1:8888/admin
```

Client subscription:

```text
http://127.0.0.1:8888/default
```

## Config

See [examples/config.example.json](examples/config.example.json).

Every client has locations. The current UI manages the first location for each client. New clients created from the UI get:

- a fresh room id generated by `olcrtc -mode gen`;
- a fresh 32-byte hex key;
- one location with selected carrier/transport/DNS.

## Reverse Proxy

The manager should usually listen on `127.0.0.1:8888` and be exposed through nginx/Caddy/Traefik with HTTPS.

Example nginx config:

```nginx
server {
    listen 9443 ssl http2;
    server_name example.com;

    ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8888;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto https;
    }
}
```

## systemd

An example unit is available at [packaging/systemd/olcrtc-manager.service](packaging/systemd/olcrtc-manager.service).

Example install:

```sh
sudo install -m 0755 olcrtc-manager /usr/local/bin/olcrtc-manager
sudo install -m 0755 olcrtc /usr/local/bin/olcrtc
sudo install -d -m 0755 /etc/olcrtc-manager
sudo install -m 0600 examples/config.example.json /etc/olcrtc-manager/config.json
sudo install -m 0600 examples/panel.env.example /etc/olcrtc-manager/panel.env
sudo install -m 0644 packaging/systemd/olcrtc-manager.service /etc/systemd/system/olcrtc-manager.service
sudo systemctl daemon-reload
sudo systemctl enable --now olcrtc-manager
```

## Security

- Do not commit real `config.json`, room ids, keys, env files, or TLS private keys.
- Use a strong `OLCRTC_MANAGER_PASS`.
- Keep the panel behind HTTPS.
- The API can start/stop network tunnel processes, so treat panel access as root-equivalent for this service.
